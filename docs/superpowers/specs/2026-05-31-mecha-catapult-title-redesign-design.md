# メカアームカタパルト + MHS撤去 + タイトル演出強化 設計書

日付: 2026-05-31

## 背景・目的

「バスケがしたいです / MHS CATAPULT」の3点を改善する:

1. カタパルトの見た目がダサいので格好よくする。
2. MHS（M/H/S）のモード切替がほとんど体感を変えないので撤去する。
3. タイトル画面を格好よくする。

ユーザー選択（2026-05-31 ブレスト時）:
- カタパルト方向性 = **ロボ/メカアーム**
- 性能統一 = **安定＆精密**（旧H相当）
- タイトル方向性 = **演出強化**（既存テキストロゴを土台に映えさせる）
- 発光色 = シアン基調でOK

すべて単一のフロントエンド構成（`index.html` / `style.css` / `script.js`、canvasドット絵）内で完結する。

## 全体パレット

- ガンメタル基調: `#2b3038`（影）/ `#3a4250`（基色）/ `#566073`（ハイライト）
- シアン発光: `#29e0e6` / `#0fb9c4`（脈動アクセント・関節・クランプ）
- 警告オレンジ: `#ff8c1a`（小アクセント、注意ストライプ）
- 既存の `#fff` / `#000` ドットUIとは引き続き共存

## 1. メカアームカタパルト

対象: `script.js` の `drawCatapult()`、`drawCatapultFlag()`、および補助関数（`drawWheel` / `drawWoodPlank` / `drawDiagPlank` は用途が変わるため新規メカ用ヘルパーに置換または追加）。

木製砲台（茶色の板・ホイール・斜め筋交い・スイングアーム）を、**装甲メカアーム式ランチャー**に置き換える。ドット絵（`fillRect` ベース）は維持。

### 構成要素
- **基部シャーシ**: 低く重厚な装甲ブロック。ガンメタル基色 + 上面ハイライト + 下面影。前面に警告オレンジの小ストライプ。
- **走行部**: 旧木製ホイールを**装甲ホイール/キャタピラ風**に置換（暗いリム + メカハブ + シアンの小点）。
- **駆動ピストン**: ピボット下に**ツイン油圧シリンダー**。シリンダー筒 + 伸縮ロッド。発射時にロッドが沈み込む（リコイル）アニメ。
- **メカアーム**: ピボットから伸びるセグメント式アーム。装甲パネルが乗り、関節とアーム先端の**電磁クランプ**がシアンに発光。
- **発光の脈動**: アイドル時も関節・クランプが `performance.now()` ベースでほのかに明滅（既存 `t` パターン流用）。
- **ボール保持**: `PHASE.AIM` 中はクランプ先端に `drawPixelBall` でボールを表示（既存と同様、座標はクランプ位置に合わせ調整）。

### アニメーション
- アーム角度は従来通り `player.angleDeg` に従う（`ctx.translate(pivot)` → `ctx.rotate(-ang)`）。
- 発射リコイル: 発射トリガ時刻からの経過で短時間（~200ms）ピストンロッド長と基部を数pxオフセットさせるイージング。実装は発射時刻を保持する変数（例: `lastFireTime`）を参照し、`drawCatapult` 内で減衰オフセットを計算（既存の `flashAlpha` / shake と同方式の自然減衰でも可）。発射タイミングは既存の発射処理（`fire`/`launch` 相当）にフックする。

### 旗
- 機能（風向き表示 + チーム表記 "T4"）は維持。
- 木ポール → **金属ポール + 小型バナー**にリスタイル。なびき計算ロジックは流用。

### 撤去するもの
- `getFocusStats()` 依存の `"MK-II"`（M固有）と `"STABILIZED"`（緑球安定）ラベル描画を削除（第2項と連動）。

## 2. MHSモード撤去

### HTML (`index.html`)
- `#screen-mhs`（CHOOSE FOCUS / M・H・Sカード / GOボタン）ブロックを丸ごと削除。
- HUDの `FOCUS` セル（`hud-top` 内 `FOCUS <span id="hud-focus">`）を削除。

### JS (`script.js`)
- STORY/FREE ボタンのハンドラ: `switchScreen(STATE.MHS)` をやめ、**直接ゲーム開始**へ（`storyMode` の true/false 設定は維持し、その後 `startGame()` を呼ぶ）。
- `getFocusStats()`: 分岐を廃し、**安定＆精密プロファイル固定**を返す:
  - `powerMul: 0.22`, `jitterAng: 0.5`, `jitterPwr: 0.7`, `guideSteps: 14`,
    `bounceRetain: 0.40`, `bankBonus: 1.2`, `predictGoal: false`,
    `stabilizeGreen: true`, `windFactor: 1.0`, `windDisplayPrecise: true`
  - （旧H相当。`guideUsesWind` は旧Sのみのキーなので含めない。）
- `mhsCards` / `btnStartGame` の取得・イベントリスナーを削除。
- `startGame()` 内の `hud-focus` 更新を削除。
- `player.focus` を参照する箇所（`script.js:362` の powerMul補正など）を確認し、focus分岐を撤去して固定挙動に。`player.focus` の初期化（`script.js:1842` 付近の `null` 代入など）は残しても害はないが、参照が消えるなら削除する。
- `switchScreen` の `STATE.MHS` ケース（画面マップの `[STATE.MHS]: 'screen-mhs'` と `AudioManager.stopBgm()` 分岐）を削除。`STATE.MHS` 定義自体も未参照になれば削除。

### 影響確認
- `player.focus` の全参照（`script.js` の 362/443-483/507-510/1352-1358 など）を洗い、固定挙動へ収束させる。HUD/ラベルの残骸を残さない。

## 3. タイトル画面 演出強化

### canvasバックドロップ
- `render()` に **`STATE.TITLE` 分岐**を追加。`STATE.GAME` 同様に星空・コート・**新メカカタパルト**・パーティクルを描画する。
  - 既存の `drawStars()` / `drawCourt()` / `drawCatapult()` を流用。タイトルでは `phase` に依存しないアイドル描画（ボール保持アニメ等は適宜）。
  - `update()` 側はタイトルではゲーム入力を処理しない（現状維持）。背景アニメ（星の瞬き・パーティクル・発光脈動）は描画関数内の時間ベースで動くため `update` 改変は最小限。
- `style.css`: `#screen-title { background: #000 }` と `.title-box { background: #000 }` を**透過**にし、canvasを透かす。テキスト可読性のため必要なら半透明の暗幕やテキストシャドウを付与。

### ロゴ・UI
- `.pixel-title`（「バスケが / したいです」）に**ネオン発光（多層 text-shadow、シアン/オレンジ）+ 脈動アニメ**。
- `.blink`（PRESS START）の演出強化。
- `.ui-button`（STORY / FREE）を**ネオンアウトライン + ホバー発光**にリスタイル。
- サブタイトル `MHS CATAPULT` は残す（ゲーム名）。
- `#best-score-display` / `.hint` 等の既存要素は維持。

## 非対象（YAGNI）
- ゲームのルール・スコア計算・ランキング・会話/ストーリー進行ロジックの変更はしない。
- 自分との闘い（dodge）モードのビジュアルは対象外。
- 新規アセット画像の追加はしない（すべてcanvasドット描画 + 既存CSS）。

## 動作確認観点
- STORY/FREE どちらからもMHS画面を経由せず即ゲーム開始する。
- ゲーム中HUDにFOCUS表示が残っていない。
- カタパルトがメカアーム外観で描画され、発射でリコイルし、関節/クランプが発光する。
- 風の旗が従来通り風向き表示として機能する。
- タイトルでcanvas背景（メカカタパルト含む）が表示され、ロゴが発光する。
- コンソールエラーが出ない（削除した参照の取りこぼしがない）。

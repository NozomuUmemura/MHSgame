## セクション1: 全体アーキテクチャ

### ゲームフロー

```
TITLE
  ├─ [START] → MHS → GAME → RESULT
  │                            └─ myRank <= 4 かつ未解放 → mhsc_unlocked="1" を保存
  │                                                        → タイトルボタンを表示
  └─ [自分との闘い] ← mhsc_unlocked="1" のときのみ表示
        ↓
      DODGE（ドッジゲーム中）
        └─ HP=0 → DODGE_RESULT → TITLE
```

### STATE 追加

```js
STATE.DODGE        = 'dodge'
STATE.DODGE_RESULT = 'dodge_result'
```

### 解放フラグ

`Storage` オブジェクトに以下を追加：

| LocalStorageキー | 内容 | 初期値 |
|-----------------|------|--------|
| `mhsc_unlocked` | 自分との闘いモード解放フラグ | `""` (未解放) |
| `mhsc_dodge_best` | ドッジモードのベスト生存時間（ms） | `0` |
| `mhsc_last_score` | 直前のカタパルトスコア（難易度係数算出用） | `0` |

`showResult()` 内で以下を実行：
1. `mhsc_last_score` に `player.score` を毎回保存（ランク問わず）
2. `myRank <= 4` かつ `mhsc_unlocked` が未設定の場合に `mhsc_unlocked = "1"` を保存
3. `updateBestScoreDisplay()` と同様に `updateDodgeButtonVisibility()` を呼び出す

`updateDodgeButtonVisibility()` は `switchScreen(STATE.TITLE)` 時にも呼び出し、  
ページリロード後も解放状態を正しく反映する。

### HTML 追加要素

- `index.html`: タイトル画面に `<button id="btn-dodge">自分との闘い</button>`（初期 `display:none`、解放時に表示）
- `index.html`: `#screen-dodge`（ドッジゲーム中の画面）
- `index.html`: `#screen-dodge-result`（ドッジ結果画面）

## セクション4: ドッジゲームのフィールドとハート

### バトルボックス

Canvasの中央に描画される白枠の矩形。フェーズが進むと変化する。

| フェーズ | ボックスサイズ（幅×高さ） | 移行タイミング |
|---------|----------------------|--------------|
| 1 | 220 × 160 px | 開始時 |
| 2 | 170 × 130 px（縮小） | 生存15秒後 |
| 3 | 260 × 190 px（拡大） | 生存30秒後 |

サイズ変化は0.5秒かけてリニア補間でトゥイーン。  
ボックス位置: Canvas中央（x=400, y=250 中心）。  
ハートはトゥイーン中もボックス内にクランプされる。

### ハートキャラクター

- **サイズ**: `fillRect` で構成した 12×12px のピクセルハート（赤 `#e61c3b`）
- **当たり判定**: 中心の 4×4px（Undertale準拠の小さめ判定）
- **被弾演出**: 白フラッシュ + 0.8秒の無敵時間（点滅）
- **初期位置**: バトルボックス中央

### 移動操作

| 入力 | 動作 |
|------|------|
| 矢印キー / WASD | 上下左右 3px/frame |
| タッチスワイプ | 移動方向に追従 |
| HUD タッチボタン | `#dodge-btn-up/down/left/right` |

移動速度: フェーズ1・3 = **3px/frame**、フェーズ2（縮小時）= **2.5px/frame**  
ハートはバトルボックスの内側にクランプされ、外に出ない。

### HUD（ゲーム中）

| 表示項目 | 位置 |
|---------|------|
| HPバー（♥ × 10、空は ♡） | ボックス上部・外側 |
| 生存時間（秒・1桁小数） | ボックス右上・外側 |
| フェーズ表示（PHASE 1/3） | ボックス左上・外側 |

## セクション5: 攻撃パターン

### 難易度係数

```js
// lastCatapultScore は直前のカタパルトゲームのスコアをLocalStorageに保存
// キー: mhsc_last_score
const difficulty = Math.min(1.0, lastCatapultScore / 200);
```

### 攻撃パターン一覧（8種）

| # | 名前 | 元ネタ | 動き |
|---|------|--------|------|
| 1 | ボール弾 | カタパルトのボール | 上から放物線で落下（黄・緑） |
| 2 | 風の矢 | 風パーティクル | 横一直線に複数本流れる（青） |
| 3 | ゴールリム | バスケゴール | オレンジの水平バーが上下から迫る |
| 4 | 直線弾幕 | 幾何学 | 4方向から直線弾が同時発射（白） |
| 5 | 回転弾 | 幾何学 | 中心から螺旋状に広がる（白） |
| 6 | 追尾弾 | 幾何学 | ハートをゆっくり追いかける1発（白） |
| 7 | 拡散弾 | カタパルトの星パーティクル | 一点から8方向放射（白） |
| 8 | 壁弾 | 幾何学 | ボックス3辺から発射、1辺に逃げ道 |

### フェーズ構成

| フェーズ | 使用パターン | 弾速 | 出現間隔 |
|---------|------------|------|---------|
| 1 | 1〜4 | `1.5 + diff × 1.5` px/frame | `1.4 - diff × 0.6` 秒 |
| 2 | 1〜6 | `2.0 + diff × 2.0` px/frame | `1.1 - diff × 0.5` 秒 |
| 3 | 1〜8 | `2.5 + diff × 2.5` px/frame | `0.8 - diff × 0.4` 秒 |

difficulty=0.0（スコア0pt）: 最低速・長間隔  
difficulty=1.0（200pt以上）: 最高速・短間隔

## セクション6: HPシステムと結果画面

### HP システム

- **初期HP**: 10
- **被弾**: -1HP + 0.8秒無敵（ハート点滅）
- **HP=0**: ゲームオーバー → DODGE_RESULT に遷移

HPバー表示（Canvas描画）:
```
♥ ♥ ♥ ♥ ♥ ♥ ♥ ♥ ♥ ♥   ← 満タン（赤）
♥ ♥ ♥ ♡ ♡ ♡ ♡ ♡ ♡ ♡   ← 3残り（空は暗い赤 #4a0000）
```

### DODGE_RESULT 画面

```
- BATTLE RESULT -

生存時間:  XX.X 秒
フェーズ:  X / 3
難易度:    ★★★☆☆

[★ NEW RECORD!]  ← ベスト更新時のみ

[RETRY]  [TITLE]
```

ベスト生存時間は `mhsc_dodge_best`（ms）として LocalStorage に保存。  
難易度の星表示: difficulty × 5 を四捨五入して ★ と ☆ で表現。

### BGM・SE

| 場面 | 対応 |
|------|------|
| ドッジ中BGM | `AudioManager.playBgm('dodge')` — 新規追加（速めの緊張ループ） |
| 被弾SE | `AudioManager.play('hit')` — 既存流用 |
| ゲームオーバーSE | `AudioManager.play('miss')` — 既存流用 |
| フェーズ移行SE | `AudioManager.play('start')` — 既存流用 |
| 結果BGM | `AudioManager.playBgm('result_good')` — 既存流用 |
